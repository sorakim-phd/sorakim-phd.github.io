project 01

Enhancing SOP Accessibility in GMP Environments

accessibility paper-track

A human-centered document UX project improving retrievability in scanned SOP archives through a compliance-safe accessibility layer rather than changes to validated files.

document UX information architecture Excel Python R usability testing

What it addressed

  • Scanned SOP PDFs were not searchable or metadata-linked.
  • Analysts relied on manual scrolling and ambiguous titles.
  • Retrieval delays increased cognitive fatigue during audits and urgent reviews.

Outcome

Built a searchable metadata layer for approximately 500 SOPs, improving retrieval speed from roughly 6–7 minutes to 1–2 minutes while preserving full GMP compliance.

project 02

Human-Centered Review Interaction in Deviation & CAPA Management

interaction paper-track

A workflow interaction project focused on review clarity, feedback consistency, and turnaround time in validated deviation–CAPA documentation systems.

workflow UX review interaction Excel Python-ready survey design latency mapping

What it addressed

  • Different QA reviewers applied inconsistent feedback styles.
  • Redundant identifier entry created friction during revision loops.
  • Small clarifications still triggered multi-day review delays.

Outcome

Developed a review interaction framework using mapping, reviewer checklists, heatmaps, and usability-oriented workflow analysis to improve clarity while preserving validation integrity.

project 03

Human-Centered Cross-Reference System for GMP Documentation

traceability case study

A metadata-driven cross-reference system improving document traceability and reducing cognitive load across deviation, CAPA, and related GMP records.

traceability metadata mapping Excel Python R workflow visualization

What it addressed

  • Validated files could not be automatically interlinked.
  • Users manually cross-referenced IDs, status, and closure records.
  • Version inconsistency and lookup burden weakened inspection readiness.

Outcome

Built an external cross-reference matrix plus computational linkage analysis, enabling faster tracking and lower version-error risk without modifying validated systems.

project 04

Cognitive Pathways in Regulated Work Environments

cognition case study

A project mapping how analysts and reviewers actually think through procedural work, revealing where human reasoning diverges from formal workflow structure.

cognitive HCI field observation interviews Excel Python network analysis

What it addressed

  • Procedural order did not always match actual reasoning order.
  • Users looped, revisited, and reinterpreted steps under pressure.
  • Bottlenecks emerged around terminology, cross-references, and validation anxiety.

Outcome

Combined observation, interviews, logging, and computational path visualization to show where procedural structure conflicts with real cognitive navigation.

project 05

Smart Feedback & Review Dashboard

analytics prototype study

A compliance-safe workflow observability project visualizing reviewer workload, latency, and bottlenecks using existing documentation metadata.

workflow observability dashboard Excel Python R latency analysis

What it addressed

  • Review cycles were opaque and often discussed only informally.
  • Workload imbalance and delay accumulation were hard to detect early.
  • Teams lacked a shared visual layer for workflow interpretation.

Outcome

Created an observability dashboard from timestamps and reviewer metadata, enabling earlier bottleneck recognition and more transparent coordination without touching validated templates.

project 06

RCA Reasoning Scaffolder

decision support paper-track

A reasoning-support project for root cause investigations, designed to help investigators explore causal hypotheses systematically instead of collapsing too early into single-cause explanations.

reasoning design causal mapping investigation support qualitative evaluation human-centered systems

What it addressed

  • RCA narratives often became linear and prematurely simplified.
  • Alternative hypotheses were difficult to sustain within existing document structures.
  • Investigators lacked an external reasoning layer to visualize causal relations.

Outcome

Designed a scaffold using investigation mapping, causal diagrams, and structured prompts to support broader hypothesis exploration while keeping validated systems unchanged.
