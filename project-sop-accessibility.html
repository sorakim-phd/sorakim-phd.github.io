SOP Accessibility + Support Layer

This project explores how procedural documentation systems shape human judgment in regulated environments. Rather than treating SOP access as simple retrieval, this prototype introduces a support layer that makes reasoning, context, and document relationships visible.

Prototype Walkthrough

Search + Retrieval Interface

This screen shows keyword-based SOP retrieval. Instead of returning documents as static results, the interface highlights matched phrases and surfaces relevant sections.

Context + Related Procedures

Beyond direct matches, the system surfaces related procedures and neighboring documentation. This addresses fragmentation across SOPs and supports cross-document navigation.

Reasoning Support Layer

This layer introduces reasoning support notes. Instead of giving answers, it keeps alternative interpretations visible and prevents premature closure during investigation.

AI-Mediated Support Layer

This prototype was extended with an AI-oriented support layer to explore how procedural systems can better support human reasoning.

The goal is not to automate compliance decisions or replace human judgment. Instead, the system focuses on making reasoning processes more visible, surfacing ambiguity, and maintaining multiple possible interpretations.

In its current form, the reasoning support is rule-based. However, this structure can be extended using large language models (LLMs).

In a future extension, LLMs could generate context-sensitive reasoning notes based on:

  • user query
  • retrieved SOP content
  • document relationships

Importantly, the role of LLMs here is not to produce final answers, but to support interpretive reasoning, highlight uncertainty, and preserve decision transparency.

Framed this way, AI becomes part of a responsibility-sensitive design approach: it expands visibility into documentation and reasoning, rather than collapsing judgment into black-box outputs.

Why this matters

In regulated environments, errors are often attributed to individuals, while underlying documentation structures remain invisible.

This project shows that improving accessibility, context visibility, and reasoning support can shift how responsibility is produced and interpreted within procedural systems.
