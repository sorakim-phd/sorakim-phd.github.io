Procedural Visibility in SOP Navigation

document UX python streamlit pilot study deployed

When scanned SOP archives block search, workers lose time before judgment even begins. This external retrieval layer shifts effort from file retrieval to document interpretation, with exploratory retrieval time reduced from ~6 to ~1 minute in pilot use.

This project argues that document accessibility is an upstream condition on judgment: retrieval burden can create interpretive burden before any formal task error appears, and before responsibility is later assigned.

retrieval difficulty → interpretive variance → judgment under constraint → later accountability

01

Problem

Procedural archives stored as scanned PDFs block keyword search and force reliance on folder memory — a structural mismatch between archive organization and real search behavior.

Observed friction points
  • Scanned files blocked keyword search
  • Titles too broad, similar, or generic
  • Users opened multiple wrong files before locating target
  • Access required archive naming familiarity
  • Time pressure amplified every friction point
Structural implication

Accessibility problems introduce hesitation and interpretive variance before any formal task error occurs. What can later look like individual difficulty may begin as structural difficulty upstream.

02

Method

01
Metadata & keyword mapping

Analyzed roughly 500 procedural documents. Extracted keywords, document identifiers, page references, and process categories to build a searchable metadata index.

02
Structure & overlap analysis

Analyzed overlapping terms, recurring categories, and keyword patterns — identifying where users face ambiguity from similar titles and mismatched labels.

03
Prototype development

Built an external retrieval layer allowing keyword-based search without altering validated source files. No modifications were made to any controlled document.

04
Pilot evaluation · think-aloud protocol

Used the prototype as both a design intervention and an analytic probe — observing how documentation access shapes confidence, search behavior, and navigation under realistic task conditions.

03

UI Prototype

Designed as an external retrieval layer operating over metadata without modifying validated files. Keyword-based access decouples retrieval from archive structure.

Constraint
Validated SOP files cannot be modified — any change requires full revalidation under GMP regulations.
Users rely on folder memory and trial-and-error file opening, creating interpretive friction before any task begins.
Decision
External retrieval layer — search operates over metadata index, not source files.
Keyword-based access decouples retrieval from archive structure while preserving document integrity.
Trade-off
No automation of interpretation or compliance decisions — human interpretive authority is preserved.
System surfaces documents; practitioners retain full responsibility for reading, applying, and judging.
04

AI Support Layer

A conceptual extension that expands the set of retrievable and comparable procedural cues, making ambiguity more visible without replacing interpretation.

Conceptual extension only — no separate pilot study was conducted. Included as a design direction rather than an empirical claim.

Open AI Layer ↗

The walkthrough below shows the AI layer expanding retrieval cues — surfacing related procedures and explaining why a result appeared, without resolving interpretation.

AI support layer · demo video

Three design principles made explicit: explains why a result appeared, surfaces related procedures, keeps interpretive options open.

Retrieval is only one component of procedural usability. Interpretive support may be necessary when the goal is to preserve reasoning openness in regulated environments.
05

Pilot Study

Participants initially relied on folder memory and trial-and-error file opening. The retrieval layer supported direct keyword-based access, with an observed reduction in retrieval time (~6 → ~1 min) under exploratory conditions. As retrieval effort decreased, participants allocated more attention to document reading rather than navigation.

N=20. Participants were recruited from the same team, which may have inflated usability and acceptance ratings. The absence of prior HCI tooling in GMP contexts also means there was no comparative baseline — participants had little to compare against. Results are exploratory and session-based; they indicate usability promise but do not establish long-term adoption, productivity effects, or downstream decision quality.

The strongest signal was not that all friction disappeared, but that ambiguity became more visible and more tractable. The prototype helped separate user difficulty from structural difficulty in the archive itself.
06

What This Revealed

Accessibility constraints shape how practitioners navigate and interpret under time pressure.

In regulated environments, retrieval is not just access — it affects which procedures are encountered, which are overlooked, and how confidently users can proceed.

Access constraints propagate into interpretive variation rather than directly determining outcomes. Accountability is not directly measured in this study; rather, the findings identify access constraints as an upstream condition shaping interpretive variation prior to downstream judgment.

07

Paper

Claim

Document inaccessibility in regulated environments creates interpretive burden upstream of formal error, and external retrieval layers can reduce that burden without altering validated files.

What it does
Develops a prototype-based study isolating accessibility as an upstream condition shaping interpretive variability in SOP systems, while arguing that usability improvements are possible without modifying validated source files.
