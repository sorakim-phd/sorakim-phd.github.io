// 01

Problem

GMP procedural documentation is a network, not a flat archive. A single deviation investigation may require navigating across SOPs, method validation documents, specification references, and training records. These documents cross-reference each other — but the links are textual rather than interactive, and the navigation burden falls entirely on the practitioner.

If we can observe how practitioners actually move through documents — where they go first, where they return, where they hesitate — we can surface structural properties of the documentation system that are otherwise invisible. Navigation behavior is a behavioral trace of documentation friction.

// observed pain points
  • Cross-references are textual ("refer to SOP-QC-004") rather than navigable — placing linking work on the practitioner
  • Related documents distributed without clear pathways between them
  • Less experienced users face significantly higher navigation burden
  • Loopback behavior is common but invisible in standard workflows
// research opportunity

Navigation behavior contains information that practitioners cannot always articulate. A loopback reveals that something in the previously visited document was not resolved — but without tracking, that signal disappears. This prototype makes those signals visible, turning implicit navigation friction into observable, structured data.

Where a practitioner loops back is not a mistake. It is a signal that the document structure failed to resolve a judgment demand the first time.
// 02

Method

01
document set design

Constructed 7 synthetic GMP documents (SOPs, analytical method protocol, reference specification) designed to reflect a realistic OOS investigation workflow — with intentional cross-referencing fragmentation to create naturalistic navigation demands.

02
scenario

Participants were given a standardised OOS scenario (ELISA assay result exceeding specification) and asked to find the relevant procedures and documents. No guidance or hints were provided.

03
behavioral tracking

The prototype automatically recorded from_doc → to_doc transitions, action type (forward / cross-reference / loopback), visit count per document, and time spent per document. Data was exported as CSV per participant session.

04
usability study · N=5

Post-task survey measured navigation clarity, cross-reference usefulness, real-world utility, and collected qualitative feedback on confusion points and improvement suggestions.

// 03

Prototype

The Cognitive Navigation Explorer presents participants with a GMP document set and an OOS investigation scenario. As participants navigate between documents, their movement is tracked in real time. At the end of the session, a results screen visualises the navigation path as a directed network graph — showing forward moves, cross-reference jumps, and loopbacks.

// 04

Usability Study

N=5 GMP-experienced practitioners, all with OOS investigation experience and daily procedural document use. Post-task survey measured five dimensions: document list clarity, content readability, cross-reference clarity, navigation awareness, and real-world usefulness.

// first document opened

4/5 participants opened an analyst-related document first — reflecting experience-based heuristics (pipetting errors are the most common ELISA OOS cause in practice). 1/5 opened the Reference Specification first to establish acceptance criteria before investigating.

// experience vs. navigation difficulty

3 participants with 5+ years GMP experience reported no confusion about navigation direction. 2 participants with ~1 year experience reported difficulty identifying which document to navigate to next — but attributed this to SOP structure ambiguity, not the prototype interface.

// 05

Findings

All five participants revisited at least one document. The most common loopback pattern: Analyst Training → ELISA Test Method SOP → Method Validation → ELISA Test Method SOP → OOS Procedure. The prototype interface was rated 5.0/5 across all dimensions — while the SOP structure was identified as the actual source of navigation difficulty.

// 06

Paper in Progress

This project is being developed into a paper on behavioral navigation tracking as a method for surfacing documentation friction in regulated environments. The paper contributes a methodological argument: navigation behavior is a valid proxy for document-structural friction — and lightweight interactive prototypes can make that behavior observable without requiring deployment inside validated systems.
