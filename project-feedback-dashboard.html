// 01

Problem

In regulated environments, document review is critical but often opaque. Deviation, CAPA, and related records may pass through multiple reviewers — yet visibility into feedback burden, timing, and revision-loop formation is limited.

As review cycles lengthen, teams often know a document is "still under review," but not what type of comments are driving the delay, where bottlenecks are forming, or whether a loop is escalating across rounds. The core problem is not delay itself, but the lack of a shared operational view of that delay.

// observed pain points
  • Repeated revision loops difficult to observe as patterns
  • Feedback burden uneven across reviewers but hard to see
  • Small wording comments could generate multi-day back-and-forth
  • Review process felt slow but source of friction remained unclear
// HCI reframing

Review delay is an observability problem as much as a coordination problem. A missing visibility layer in the workflow meant that friction accumulated without ever becoming legible enough to act on early.

// 02

Method

01
dataset construction

Created a synthetic review-workflow dataset (100 records) representing comment categories, review rounds, severity, turnaround time, reopening, reviewer role, and document type.

02
UI prototype — 6-tab dashboard

Built an interactive dashboard in Python/Streamlit: workflow summaries, revision heatmaps, checklist assistant, rule-based comment classifier, document journey tracing, and reviewer simulation.

03
AI extension — TF-IDF clustering

Added TF-IDF similarity clustering for recurring issue detection, multi-factor revision burden scoring, and pattern summaries — all as observability tools rather than decision-automation systems.

04
usability feedback · N=6

Informal prototype feedback session with six QA/QC-experienced users to assess clarity, interpretability, interface organization, and perceived usefulness.

// 04

AI Extension

The AI extension adds a computational pattern-detection layer on top of the rule-based dashboard. Rather than automating reviewer judgment, it uses TF-IDF similarity clustering to detect recurring issue patterns across review rounds — surfacing structural friction that is invisible in individual comment threads.

Grounded in the same design principle as the UI prototype: AI as a visibility tool, not a decision-replacement. It makes hidden patterns legible without collapsing reviewer discretion into algorithmic output.

// what AI does
  • TF-IDF similarity clustering — auto-detects recurring patterns without manual tagging
  • Revision burden scoring — quantifies cognitive and coordination load per document
  • AI-like pattern summary — plain-language description of review state and bottleneck stage
// what AI does not do
  • Does not recommend actions or assign blame
  • Does not replace reviewer judgment
  • Does not modify validated documentation systems
  • Surfaces patterns for human interpretation only
AI surfaces patterns for human interpretation — it does not determine what should be done about them.

// 06

What This Revealed

The main contribution is not efficiency. It is the transformation of hidden review work into interpretable workflow signals. The prototype supports procedural visibility and coordination without automating reviewer judgment.

The key insight from the feedback session: participants didn't struggle with the interface itself. The workflow-friction concept initially felt unfamiliar — pointing toward adoption barriers being conceptual rather than interactional. The next design challenge is not interface clarity, but how to make the value of friction visibility legible to users trained to absorb review load silently.

hidden friction → visible pattern → earlier sense-making → judgment supported
// 07

Paper in Progress

This project currently functions as a portfolio-facing HCI case study. Its main contribution is to show how review friction, comment burden, and revision loops can be made visible through a lightweight computational support prototype — and how AI can be applied as a pattern-detection layer rather than a decision-automation system.
