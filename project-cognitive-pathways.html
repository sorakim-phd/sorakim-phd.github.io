🧩 Background

problem framing

In Good Manufacturing Practice (GMP) environments, analysts and reviewers are expected to follow highly structured documentation and procedural workflows. On paper, these workflows appear orderly, linear, and complete.

In practice, human reasoning is rarely linear. People pause, revisit earlier sections, compare multiple instructions, cross-check prior records, and reconstruct meaning in context while trying to stay compliant.

This creates a tension between how procedural work is formally written and how cognition actually unfolds during use. That tension also connects this project directly to my Cognitive Friction paper, which argues that procedural systems often externalize interpretive burden onto the person following them rather than resolving that burden in the document itself.

💬 Human reasoning often unfolds through returns, comparison, hesitation, and repair, while procedural documentation is written as if cognition proceeds in a straight line. This project makes that mismatch visible at the level of reasoning pathways, while the CF paper explains it as a structural property of procedural writing.

⚠️ Observation

cognitive friction

Observed pain points

  • Users moved back and forth across sections rather than simply following written order.
  • Repeated cross-checking was needed around terminology, prior records, and ambiguous instructions.
  • Reasoning often depended on reconstructing context rather than merely executing a written step.
  • Interruptions and detours made it difficult to maintain the same cognitive state across the workflow.

Why this was difficult

In regulated environments, the challenge is not just to complete the procedure, but to absorb the interpretive work that the procedure leaves unresolved. The issue was not "users failed to follow the workflow," but rather "the workflow demanded more cognitive repair than the written structure acknowledged."

🎯 Design question

HCI framing

How can cognitive pathways in regulated documentation work be reconstructed so that hidden interpretive burden, hesitation, and pathway mismatch become visible as design problems rather than being treated only as user difficulty?

🧠 Method

computational + qualitative

1. Synthetic pathway dataset construction

Generated a synthetic dataset representing transition behavior in regulated documentation work, including forward movement, cross-reference behavior, loopbacks, and hesitation-like pathway patterns.

2. Cognitive pathway mapping

Translated documented work sequences into node-edge structures where nodes represent procedural locations and edges represent actual cognitive transitions across those locations.

3. Transition and frequency analysis

Used Python and pandas to analyze transition structure, distinguish forward versus non-forward movement, and identify where pathway irregularity and repeated returns concentrated.

4. Interactive demonstrator

Built a Streamlit-based prototype to visualize pathway structure, transition patterns, and friction-relevant behavior so that cognitive flow could be explored as a structural property of procedural work.

🧪 Evaluation logic

prototype evaluation

Evaluation setup

Because the project's main contribution is conceptual and analytical, the prototype was treated as a demonstrator rather than a production tool. The focus was on whether pathway structure made hidden cognitive work more visible.

workflow tracespathway modelstructural visibility

What was examined

  • whether non-linear pathway behavior became structurally legible
  • whether hesitation and return paths could be distinguished from simple compliance flow
  • whether transition structure clarified where documentation imposed extra cognitive burden
  • whether the prototype made the broader CF argument more tangible

Why this matters

The point was not to challenge GMP compliance. It was to ask a more human-centered question: if pathway structure is surfaced clearly, can we better understand where documentation itself creates unnecessary cognitive work? In that sense, the project supports the broader CF argument that procedural systems can appear complete on paper while leaving substantial interpretive labor to the human actor.

💬 Main takeaway

The value of the project is not just that it visualizes movement through documentation, but that it turns invisible reasoning burden into something structurally inspectable. It makes pathway-level evidence available for a design argument about cognitive friction.

📄 Paper in progress

research output

This project is being developed into a paper on cognitive pathways, procedural mismatch, and human reasoning under regulatory constraint. It also sits in direct conversation with my Cognitive Friction paper: where the CF paper develops the theoretical argument that procedural texts generate hidden interpretive burdens, this project traces how those burdens appear in actual reasoning pathways during regulated work.

Together, the two pieces form a linked argument. The CF paper explains friction at the level of procedural structure and wording; this project shows how that friction surfaces behaviorally as branching, hesitation, return paths, and non-linear navigation under documentation pressure.

💬 Reflection

why it is HCI

This project reframes documentation work as a cognitive systems issue rather than a simple matter of whether users comply correctly. In other words, pathway irregularity is not always a sign of weak performance; it can also be evidence of systems that demand hidden repair work from the human mind.

From an HCI perspective, the important insight is that documentation should be evaluated not only for formal completeness but also for cognitive realism. It also reinforces the broader claim of the Cognitive Friction paper: when procedural systems fail to carry their own interpretive load, that burden is shifted onto human reasoning.
